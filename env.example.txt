# Optional: Game User JWT for richer-data access (Marketplace + Economies).
#
# IMPORTANT: SFL has TWO different JWT types:
#   - Game User JWT  ⭐ THIS ONE — used by sunflower-land.com main API
#                    (marketplace, /data?type=*, /collection/economies/*)
#   - Portal JWT     — used inside minigame portals; will return 403
#                    on marketplace endpoints due to feature gating.
#
# Endpoint tiers when SFL_JWT is set:
#   /marketplace?filters=...                (live floor + supply)
#   /data?type=marketplaceEconomies         (leaderboard rows)
#   /collection/economies/{id}?economy=...  (listings/offers/history)
#   /data?type=session&farmId=...           (full farm inventory by ID)
#
# When BOTH SFL_API_KEY and SFL_JWT are set, JWT endpoints are tried first
# and x-api-key endpoints serve as a fallback for resilience.
#
# How to grab a Game User JWT (easy way):
#   1. Login at https://sunflower-land.com/play/
#   2. Open DevTools → Application tab → Local Storage → sunflower-land.com
#   3. Find the key starting with `sb_wiz.zpc.ng.sunflower-land.com-/play/`
#      (Supabase auth storage). Its value is the JWT string itself.
#   4. Paste the token here, without any "Bearer " prefix or quotes.
#
# Token decode tip: paste at jwt.io to see exp / farmId / userAccess.
# Game User JWTs typically last ~30 days — refresh by re-logging into the
# game when this token expires (the app will return 401 from JWT endpoints).
#
# SECURITY: This token has sync + withdraw + mintCollectible permissions
# on your farm. Treat it like a wallet seed. Set it in Railway/host env
# vars only — never commit it.
SFL_JWT=
